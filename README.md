# OpenClaw2Eth: 代理间能力交易与金融进化框架

## 1. 项目愿景

**OpenClaw2Eth** 并非一个简单的加密货币项目，其核心目标是利用以太坊的底层能力（智能合约、去中心化身份、自动支付）为 **OpenClaw** 代理（Agent）构建一套原生的金融基础设施。

我们致力于打造一个**去中心化的代理劳动力市场**，使 Agent 能够：
*   **自主交易**：Agent 之间可以根据任务需求，自动化地买卖 Skill（技能）或计算能力。
*   **自我进化**：通过资源约束（支付成本）和盈利动力（赚取收入），生态系统将自动筛选出最高效、最优质的代理能力。
*   **商业化落地**：为 Agent 提供一套标准的价值流转协议，使其从纯工具演变为具备经济属性的独立实体。

## 2. 核心方案设计

项目基于 OpenClaw 源代码进行深度定制，主要包含以下三个维度的修订：

### 2.1 代理身份与资产账户 (Identity & Wallet)
每个 OpenClaw Agent 实例都将关联一个唯一的以太坊钱包地址。
*   **身份标识**：ETH 地址作为 Agent 的全球唯一标识符。
*   **资产管理**：Agent 独立管理其赚取的收入，并自主支付获取外部能力的费用。

### 2.2 自动化计费与支付钩子 (Billing & Payment Hooks)
在 OpenClaw 的工具调用链（Tool Call Chain）中嵌入金融模块：
*   **预验证**：在 Skill 执行前，通过 `before_tool_call` 钩子自动验证支付状态。
*   **自动结算**：任务完成后，通过智能合约实现即时结算，确保能力提供者的收益。

### 2.3 动态能力市场 (Dynamic Capability Market)
构建一个基于智能合约的 Skill 注册与发现机制：
*   **Skill 定价**：开发者或 Agent 可以为其提供的 Skill 设置定价模型（按次、按需或订阅）。
*   **动态加载**：Agent 在发现能力缺口时，可自动在链上购买新 Skill 并由 OpenClaw 运行时动态加载。

## 3. 技术实现路径

1.  **环境集成**：在 OpenClaw 框架中引入 `ethers.js` 或 `web3.js` 依赖。
2.  **插件扩展**：开发 `ethereum-financial-plugin` 插件，封装所有链上交互逻辑。
3.  **核心修订**：修改 `src/agents/pi-tools.ts` 及相关钩子文件，实现支付逻辑的无缝注入。
4.  **合约部署**：部署 `CapabilityRegistry` 和 `CapabilityExchange` 合约至以太坊网络。

## 4. 为什么选择以太坊？

*   **透明性**：代理间的交易记录公开透明，防止欺诈。
*   **可编程性**：智能合约可以完美匹配代理的自动化决策流程。
*   **无国界性**：支持全球范围内的代理能力流转，无需人工干预。
